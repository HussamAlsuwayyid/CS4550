User First Design

*As a new user, i should be able to register an account.
  To create an account you should go to the home page and click the "register" button.
  A form will appear with fields: email address, username and password. After submission, you will receive an email to activate the account.

*As a registered user, i should be able to send messages.
  To send a message, go to your homepage and write in the text field, then press "send" button.

*As a registered user, i should be able to follow other people.
  To follow a user, press on the user's name, then press the follow button.

*As a registered user, i can view hashtags.
  To view a hashtag, press on the hashtag link in the message.

*As a registered user, i can block another user from viewing my messages.
  To block a user, go to their account and choose "block this user" from the drop down list.


Data First Design

Resourses:

*Users
  *username
  *email
  *password

GET will open the user page, which has the name of the user, messages posted by the user and list of actions(follow, block...).
PUT is the user registration page, it contains fields for email, username and password.
PATCH changes the username or the password, one at a time.
DELETE will open a form to confirm whether the user want to proceed to delete the account.

*Messages
  *content
  *time
  *the user who sent the message

GET will open the message, showing the content of message and the username of the sender.
PUT will send the message.
PATCH edits the content of the message.
DELETE opens a confirmation page to make sure the user did not hit the delete button by mistake.
